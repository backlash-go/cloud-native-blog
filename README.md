项目 一   基于企业的认证权限管理SSO(单点登录)

项目背景（Why）

公司运维体系涉及 Grafana（监控）、JumpServer（堡垒机）以及阿里云、腾讯云等多云平台，各系统采用独立认证机制，缺乏统一身份管理。

运维人员在日常操作中需要频繁切换登录多个系统，认证流程重复且效率低下。同时，各平台用户与权限分散管理，账号生命周期无法统一控制, 入职离职需要开通或者删除清理大量账号，存在权限冗余、越权访问及审计困难等问题。

在多云架构下，权限控制与安全合规要求进一步提高，传统分散式认证已无法满足集中化运维与安全管理需求。

设计并实现统一 SSO（基于 OAuth2.0 / JWT / SAML 等协议），整合 Grafana、JumpServer 及云平台登录体系，实现统一认证、统一授权及集中审计能力。

使用IAM 平台 后续扩展到运维平台以及K8S等等支持OIDC SAML主流协议认证的系统。


职责与工作内容（What）

调研主流 SSO 解决方案，分析不同认证协议的适用场景及实现原理包括 auth2.0 smal协议等
对比开源与商业化 SSO 产品的功能差异、部署成本及扩展性（如 Keycloak、zitadel 等） 开源化项目主要问题是部分应用例外最新版本jumpserver针对oidc认真需要收费，商业化项目收费比较昂贵
各类IAM平台支持的协议大致都相同，最终选择开源免费keycloak项目 主要因阿里云这种公有云ram登陆不支持SP-Initiated认证登陆


项目成果：
- 实现统一身份认证平台（基于 Keycloak），打通 Grafana、JumpServer 及多公有云账号阿里云/腾讯云等多平台登录体系
- 支持 OAuth2.0 / OIDC / SAML 等主流协议，兼容多种系统接入（含 K8s、运维平台等）
- 实现 SP-Initiated 与 IdP-Initiated 双模式登录，提升用户访问灵活性

- 运维人员登录效率显著提升，减少多系统重复登录（登录操作减少约 60%+）
- 账号生命周期实现统一管理（入职/离职自动开通与回收），人工操作成本降低约 70%
- 权限集中管控，减少权限冗余与越权风险，提升整体安全性与合规性







项目一：企业级统一认证与权限管理平台（SSO）
项目背景（Why）

公司运维体系涉及 Grafana、JumpServer 以及阿里云、腾讯云等多云平台，各系统采用独立认证机制，缺乏统一身份管理。

运维人员需频繁登录多个系统，认证流程重复且效率低下；同时用户与权限分散管理，账号生命周期（入转调离）无法统一控制，存在权限冗余、越权访问及审计困难等安全风险。

在多云架构下，安全合规与权限管控要求提升，传统分散式认证体系已无法满足集中化运维需求。因此设计并实现统一 IAM / SSO 平台，基于 OAuth2.0 / OIDC / SAML 等协议，打通多系统认证，实现统一身份认证、权限控制与审计能力，并支持后续扩展至 K8s 及运维平台。

职责与工作内容（What）
调研主流 SSO/IAM 方案，分析 OAuth2.0、OIDC、SAML 等认证协议的实现机制及适用场景
对比开源方案（Keycloak、Zitadel）与商业化 IAM 产品在功能、成本及扩展性方面的差异，完成技术选型
结合 JumpServer OIDC 收费限制及公有云（如阿里云 RAM 不支持 SP-Initiated）等约束，制定可落地的统一认证方案
基于 Keycloak 搭建统一身份认证平台，设计多系统接入架构，支持 Grafana、JumpServer 及多云平台统一登录
实现基于 OIDC / SAML 的标准化接入流程，支持 SP-Initiated 与 IdP-Initiated 双模式认证
构建统一账号与权限管理体系（RBAC），实现用户生命周期（入职/离职）自动化管理

项目成果（Result）
搭建企业级统一 IAM / SSO 平台（基于 Keycloak），打通 Grafana、JumpServer 及阿里云/腾讯云等多平台认证体系
支持 OAuth2.0 / OIDC / SAML 多协议接入，具备良好扩展性，已适配 K8s 及运维平台等系统
实现 SP-Initiated + IdP-Initiated 双登录模式，覆盖多种业务访问场景
运维效率显著提升：多系统登录流程统一，登录操作减少约 60%+
账号生命周期统一管理：入职/离职自动开通与回收，人工操作成本降低约 70%
安全能力提升：权限集中管控，有效减少权限冗余与越权访问风险
构建统一审计能力，实现跨系统登录行为可追踪，满足安全合规要求
基于开源方案替代部分商业 IAM 产品，降低系统建设与运维成本
